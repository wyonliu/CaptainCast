# EP.002 播客脚本
# 主题：造宇宙的第一天——报错、崩溃与顿悟
# 副题：我们用 AI 做这个节目的过程，就是这个节目本身
# 时长：约10分钟 / 22段

---

[船长 speed=0.95] 欢迎来到CaptainCast，我是船长。今天这期节目，有点特别。它是用一套全自动系统生成的——AI合成的声音，AI生成的配图，自动上传到公众号。但更特别的是，这期节目的内容，就是我们造这套系统的过程本身。

[麦洛 speed=1.05] 等等爸，我没听懂。你是说，这个节目……是在说它自己怎么被做出来的？

[船长 speed=0.92] 对。你听到的这两个声音，是用你真实的声音和我真实的声音，训练出来的克隆体。而你现在听到的这期节目，是由那套系统，自动合成的。

[麦洛 speed=1.10] 那……是真正的爸爸在说话吗？

[船长 speed=0.88] 是真正的爸爸写的稿子，AI的嘴巴说出来。就像《神临山海》里的山海世界——麦洛是真实米莱的投影，船长是真实我的投影。这个节目，是我们父女的数字孪生。

[船长 speed=0.95] 今天，2026年3月4日。从下午开始，我和一个叫小安的AI，用大概十几个小时，把这套系统从零搭出来。我想跟你说说，这一天发生了什么。因为，这一天里发生的每一个"报错"，都印证了上一期说的那个理论——灵机。

[麦洛 speed=1.05] 报错怎么能是灵机？报错不是应该修掉吗？

[船长 speed=0.90] 正常逻辑是：遇到问题，解决问题，推进项目。但每一个报错，其实都是宇宙给你塞进来的一个新变量。我们遇到的第一个问题——AI生图模型的名字写错了。

[麦洛 speed=1.08] 就是……打错字了？

[船长 speed=0.90] 不是打错，是按照文档写的，但那个模型刚改了名字，文档还没更新。于是小安去查了一遍所有可用模型，发现正确的名字叫——别管那串英文了，总之找到了。但这一找，顺手发现那个模型是"文字生图竞技场排名第一"。如果不报错，我们不会去查，就不会选到最好的那个。

[麦洛 speed=1.00] 哦，报错反而选了更好的？

[船长 speed=0.88] 然后是第二关。中国的IP被限制了，Gemini的图像生成服务不对大陆开放。小安发现了一个本地代理，绕过去了。然后是第三关——音频。

[麦洛 speed=1.05] 音频出了什么问题？

[船长 speed=0.85] 大问题。微信公众号的文章可以嵌入音频，但是通过API发布的时候，我们用了一个叫`mpvoice`的标签，它一直报错：无效内容。我们试了三个小时，换标签格式、换上传方式、换文件大小。

[麦洛 speed=0.95] 然后呢？

[船长 speed=0.85] 然后我说了一句话：你想想办法，我见过很多有音频的公众号。小安去把那篇已经被我手动插入了音频的草稿，重新读了一遍——用API读回来，看里面的HTML源码。

[麦洛 speed=1.10] 然后发现了什么？

[船长 speed=0.82] 真正的标签，不是`mpvoice`。是`mp-common-mpaudio`。一个从来没在任何官方文档里出现过的标签。里面藏着一个字段：voice_encode_fileid，解码出来是——三十二亿…加一串数字。那是这个公众号账号的内部ID，加上文件序号。

[麦洛 speed=0.90] 爸，这听起来像解密。

[船长 speed=0.85] 就是解密。我们花了大半天，最终找到了微信服务器里，音频文件真正的标识方式。然后把这个标签重新写进系统，再跑——草稿更新成功。音频在文章顶部，就是你现在听到的这个。

[麦洛 speed=1.05] 所以……我们现在听到的这期节目，就是用那套系统做的？

[船长 speed=0.88] 对。你说了一句"哦，报错反而选了更好的"，那个声音，是MiniMax用你30秒的真实录音，克隆出来的。我写了脚本，小安生成了声音，生成了配图，上传到了公众号草稿，音频也嵌进去了。

[麦洛 speed=1.00] 那爸……这算灵机吗？

[船长 speed=0.85] 你看，理性的最优解是：遇到音频问题，换一个不需要音频的方案。但我没有放弃，坚持要原生音频。这不理性。然而正是这个不理性，逼出了那个隐藏标签。这一天里每一个"不对、重来"，都给宇宙多加了一个新变量。最后这个系统，比我们最开始设想的要强得多。

[麦洛 speed=0.95] 那下一期，是什么？

[船长 speed=0.85] 下一期，我想聊灵机共鸣——当一百万人同时发出非理性的呐喊，宇宙会发生什么？但那是另一个故事了。欢迎来到CaptainCast，关注我们，下一期见。
