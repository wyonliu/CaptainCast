## 图2：技术揭秘图（配图：mp-common-mpaudio标签特写+解码过程）

**标题：微信音频的秘密：一个藏在源码里的标签**

今天花了3小时，
被同一个报错拦住：

```
{"errcode": 45166, "errmsg": "invalid content"}
```

官方文档说用 `<mpvoice>` 。
用了，报错。

最后我们把一篇已经有音频的草稿读回来，
看它的原始HTML——

发现了一个叫 `mp-common-mpaudio` 的标签。
里面有一串Base64编码：
解码出来是账号内部ID + 文件序号。

这就是微信真正的音频系统。
从来没有人公开写过它。

我们是逆向出来的。
